# 拆链工作流（/拆链 赛道名）

目标：以**海外龙头为入口逆向拆解**赛道产业链 → 梳理完整链条 → 找出**产能瓶颈卡口** → 映射 A 股标的 → 四问打分 → 出「卡口清单」。

## 输入
- `赛道名`：用户指定的赛道（如 算力 / 人形机器人 / 半导体）

## 工作目录（每次执行独立）
每次调用在 `runs/<赛道>/` 下工作，所有产物收纳在此：
- `runs/<赛道>/*.py`：本次取数脚本
- `runs/<赛道>/data/`：拉取的一手资料（filing/PDF/文本）
- `runs/<赛道>/chain.yaml`：产业链树
- `runs/<赛道>/卡口清单.md`：最终输出
公用库（`astock.py` / `sec.py`）留在根目录，脚本里 `import` 即可。

## 步骤

### Step 1 — 扫热点选锚 + 拉取研读一手资料
1. **多角度、多轮 web 搜索广扫近 2 日海外英文源**，定位赛道**新冒头的焦点**作为**锚**：
   - **广度优先**：从多个角度（新品/平台、产能/涨价/缺货、龙头动作/电话会、技术路线切换等）广撒网，目的是**不漏掉刚出现的新信号**。
   - **选锚 = 新出现 / 边际加速、尚未成为共识的焦点**：反复出现的老热点是市场已确认、已 price in 的，做锚就晚了；要抓的是刚冒头、还没被反复报道的。
   - **过滤噪音**：新信号需有**实体支撑**（指向具体产品/供应链事件/龙头动作），而非纯观点或预测；既不取已饱和的老共识，也不取无实体的单篇头条。
   - **只看海外英文源、不看中文新闻/研报**；热点仅用于选锚，不用于判卡口。
2. **拉取锚定产品及其龙头的一手资料**（SEC filing / teardown / 拆解 BOM / 行业资料；工具见 `sec.py`）并**研读**。
3. 资料存 `runs/<赛道>/data/`，热点来源与锚定产品记入 `source_refs`。

### Step 2 — 构建产业链树 + 找产能瓶颈（纯结构，不挂标的）
1. **结合 Step1 资料 + 自身行业知识，逐层构造完整产业链结构**（拆链本就是资料与认知结合；但区分清楚哪些有资料支撑、哪些是推断）。
2. **逆向拆锚定产品的完整 BOM，钻到料件/材料层**：子系统 → 环节 → 关键材料/被动件/设备，**不停在系统件**（藏在系统件里的料件也要拆出来）；先不预判卡口。
3. **在完整链上找产能瓶颈卡口**：哪个环节卡住全链产出、扩不上来、别的环节在等它。依据龙头电话会/研报反复点名的**紧缺、涨价、交期拉长、扩产周期长**等信号；命中的叶子标 `bottleneck: true` + 一句话依据。
4. 叶子标 `global_supply`（全球供应格局）/ `region`，**不挂标的**。
5. 产出 `runs/<赛道>/chain.yaml`（tickers 留空），按下方 schema；交用户审树后再进 Step 3。

### Step 3 — 四问评分（按下方评分卡，逐环节打分）
- 寡头、没人看：用脚本量化（脚本放 `runs/<赛道>/`，调 `astock.py`）。
- 能替、能扩：用研报+财务辅证指标判断，给出理由。
- 每问 1/3/5 分，记录依据。

### Step 4 — 出卡口清单
- 综合得分 = 四问加总（满分20）。
- 按得分降序输出至 `runs/<赛道>/卡口清单.md`（Markdown 表）。
- 高亮「绕不开(能替=5) + 扩不了(能扩=5) + 寡头(=5) + 没人看(=5)」的环节。

## chain.yaml schema（第3↔4步契约）

> schema 支持**任意层级与分支**：每个 node 可有 `children`，可递归嵌套，不限层数、不限命名维度。
> 只有「叶子环节」（无 children、可定位标的的最细环节）才需要挂 `tickers` 和参与四问打分；中间节点用于组织树结构。

```yaml
sector: 算力                 # 赛道名
updated: 2026-06-10          # 拆链日期
anchor_leader: 英伟达 GB200 NVL72   # 海外龙头入口（逆向拆解起点）
source_refs:                 # 多源依据(招股书/10-K/海关/专利/研报)
  - "英伟达 FY25 10-K"
  - "XX证券：AI算力产业链深度"
tree:                        # 产业链树（任意嵌套分支，骨架来自龙头BOM）
  - name: GPU/算力芯片        # 来自龙头BOM的子系统
    dim: 芯片
    children:
      - name: 先进制程代工
        layer_type: 制造
        global_supply: 台积电垄断, 国产中芯追赶   # 全球供应格局
        a_share: 国产替代
        tickers:             # 叶子环节才挂标的池 + 参与打分
          - {code: "688981", name: "中芯国际"}
      - name: ArF光刻胶
        layer_type: 材料
        physical_choke: 树脂分子配方+ppb级超纯, 物理+认证双壁垒
        global_supply: JSR/信越/东京应化寡头>80%
        tickers:
          - {code: "300346", name: "南大光电"}
      - name: 先进制程光刻机
        layer_type: 设备
        region: 海外          # 海外环节, 仅参照不打分
        a_share: 无
        physical_choke: EUV光源+蔡司物镜, 仅ASML
```

## 四问评分卡 rubric（固定尺度，避免漂移）

> **统一方向：分数越高 = 越像卡口 = 越值得投。** 满分 5，最低 1。

### Q1 不可替代性（原「是否能替」，分越高越替不掉）
- 5分：技术/专利/客户认证壁垒极高，国内外几乎不可替代
- 3分：有一定壁垒但存在潜在替代者
- 1分：标准化、易被替代
- 辅证指标：毛利率、研发费用占比

### Q2 扩产难度（原「是否能扩」，分越高越扩不了）
- 5分：扩产周期长/资本开支巨大/设备人才稀缺，短期扩不了
- 3分：扩产有门槛但可逐步推进
- 1分：产能容易快速扩张
- 辅证指标：在建工程、资本开支、产能利用率

### Q3 是否是寡头？（集中度）
- 5分：CR3 ≥ 70% 或明显寡头垄断
- 3分：CR3 40%~70%，集中度提升中
- 1分：CR3 < 40%，分散竞争
- 量化来源：环节内标的营收/市占率算 CR3/CR5

### Q4 是否没人看？（关注度，越冷越高分）
- 5分：研报覆盖家数少 + 换手率分位低 + 估值分位低（冷门）
- 3分：关注度中等
- 1分：研报扎堆 + 换手率高 + 估值高（拥挤）
- 量化来源：研报覆盖家数、换手率分位、PE/PB 分位

## 输出格式（卡口清单）

```markdown
# <赛道> 卡口清单（生成日期）

| 环节 | 能替 | 能扩 | 寡头 | 没人看 | 综合 | 代表标的 | 卡口理由 |
|------|-----|-----|-----|-------|------|---------|---------|
| 衬底 | 5 | 5 | 5 | 5 | 20 | XX | 技术垄断+扩产难+冷门 |
```
- 综合得分降序排列，20分项为最高优先级卡口。
